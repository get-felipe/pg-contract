# Code of Conduct

This project uses the Contributor Covenant as its baseline for community behavior. The goal is a practical, respectful project space where people can report bugs, ask questions, propose changes, and review code without personal attacks or harassment.

## Expected Behavior

- Be respectful and professional.
- Focus critique on the work, not the person.
- Assume good intent, but accept correction when impact and intent differ.
- Keep discussions relevant to the project.
- Give clear technical context when disagreeing.

## Unacceptable Behavior

- Harassment, intimidation, threats, or sustained disruption.
- Insults, personal attacks, or discriminatory language.
- Publishing private information without explicit permission.
- Sexualized language or imagery in project spaces.
- Repeatedly derailing issues, pull requests, or discussions after maintainers ask for focus.

## Scope

This code applies to project spaces including issues, pull requests, discussions, reviews, release comments, and community channels controlled by the project. It also applies when someone is clearly representing the project in a public space.

## Enforcement

Maintainers may edit, hide, or remove comments; close or lock threads; reject contributions; or temporarily or permanently limit participation when behavior violates this code.

Enforcement should be proportional, documented when practical, and focused on keeping the project usable for contributors and users.

## Reporting

Report conduct concerns privately to the maintainer when possible. If no private channel is available yet, open a minimal public issue asking for a private contact path and do not include sensitive personal details.
